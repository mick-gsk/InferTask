# Contributing — InferTask

InferTask ist ein Lernprojekt — Solo-Entwicklung von Mick Gottschalk.

---

## Wie Issues abgearbeitet werden

1. Issue vollständig lesen (Ziel, Scope, Acceptance Criteria, Verifikation)
2. Nur die unter **Scope** genannten Dateien ändern
3. Alle **Acceptance Criteria** manuell prüfen
4. **Verifikationsschritte** im Browser durchführen
5. Committen mit passendem Commit-Typ (siehe DEVELOPER.md)
6. Issue schließen

---

## Regeln

- Kein ungefragtes Refactoring
- Kein `any` in TypeScript ohne Kommentar
- Keine externen Bibliotheken ohne Freigabe
- Code auf Englisch, Kommentare auf Englisch
- Keine Cloud-APIs

---

## Neue Issues melden

Bitte das Issue-Template **Arbeitsauftrag** oder **Bug** verwenden.
Leere Issues ohne Template werden nicht bearbeitet.
